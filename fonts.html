Fonts

Choosing the correct fonts for your webpage can be a taxing endeavor. Tuxedo makes it easier with some font pairing that go great together. Simply follow the steps below to change the fonts throughout your project.

Making font changes

Tuxedo's _global.scss file contains all the variables that dictate the fonts of your web app. When you open the globals file you should see the following (for our current purposes we are going to ignore everything except the font section of the file):



(...) //fonts - to update, import a new font below @import url('https://fonts.googleapis.com/css?family=Josefin+Sans:300,400,700,700i|Lato:300,400,700'); //Define font family for header and body font. Must already be imported above $header-font: 'Josefin Sans', sans-serif; %body-font { font-family: 'Lato', sans-serif; font-weight: lighter; letter-spacing: .5px; } (...)

If you've never used Sass before this might look a little mysterious so let's break it down. Sass allows you (the developer) to create variables that are implemented throughout your CSS. Variables are defined with a $ preceeding them. Tuxedo's variable names are descriptive so you know exactly what you're changing. Now, wherever you see $header-font throughout your SCSS files, you know that is actually a placeholder for the header-font you've defined in your global styles.

Notice how body-font has a % in front of it's name. This is called a placeholder selector. Placeholder selectors are typically coupled with the @extend Sass directive and allow the user to place blocks of styling that can then be "extended" to other elements. This placeholder selector can be used by writing @extend %body-font when styling. Check out the _fonts.scss file to see it at work.

Finally, if you want to change font-styling such as padding, margin, font-size, etc. to headers or paragraphs visit the _fonts.scss file. Just remember, changes made here will reflect across your entire app.
