Tuxedo was designed with you - the developer - in mind.

Whether you are front-end or back-end, this clearly commented and semantic framework will get your CSS up and running in no time. Dig into our Sassy CSS (SCSS) files until your site is fully customized - we’ll tell you how.

Getting Started

Full version of Tuxedo

To get started - download the full version of Tuxedo. This version comes complete with all the JavaScript and SCSS you need to get your project underway. Each SCSS file is well commented so you'll know what parts of the code to change to customize the appearance of your app.

Setting Up Sass

These instructions are for setting up on a Mac. If you are on Windows or Linux read the documentation for getting Sass installed on your machine here.

Navigate into the tuxedo-full folder you downloaded. To get started, we are going to set a Sass watch on the the SCSS components of the project. This will compile any changes you make in the SCSS files into CSS the browser can read. If you already have Sass installed on your machine ignore the first snippet below and just run the watch in terminal. Once you are done, open a new terminal window or tab in order to leave the watch running. The Sass watch should be left running anytime you are working on the project making changes to the SCSS files.

gem install sass
sass --watch scss:css

Minified version of Tuxedo

Alternatively, you can download just the minified CSS and JavaScript version of Tuxedo. This is a good option for developers who don't want to touch any of the SCSS files and want to use the styles straight out of the box. The drawback to this method is that you will have to use your own stylesheet to override any styles you want to change. No need to run the Sass watch above with this version.

Download One Component at a Time

If you only want to download Tuxedo one component at a time, click on a link below to get started. This option is great for those looking for a grid, forms, etc. who don't want to avoid large file sizes. Each of these downloads comes with the necessary SCSS files, a base index.html, as well as any necessary JavaScript files. Don't forget to run the Sass watch described above for you SCSS files to be compiled as CSS.

Project Structure

If you've downloaded the full version of tuxedo - either from the download button or by cloning the repo - here is a look at what your Tuxedo folder contains.

The index.html page contains the basic project structure to get your website started. Tuxedo is dependent on jQuery for its JavaScript to run, so you'll notice the jQuery CDN in the head tag of your index.

The tuxedo.js file has everything you need to make things like nav bars and animations work. There is a section at the bottom of the file for you to place function calls to make the components you need work.

There is an img folder with great high-res images from Unsplash to get your project started.

Finally, there is the scss folder full of all the components you need for your project. Unlike other frameworks, Tuxedo is design so you can directly edit these files. Code is clearly commented and easy to follow. Start off in the _global.scss folder where you can decide what colors and fonts to use. Tuxedo is designed to have several neutral colors and then one pop of color to keep your site looking clean and beautiful.
